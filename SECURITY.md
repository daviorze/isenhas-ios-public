# Security Policy

## Supported Scope

This repository contains the public client-side implementation of the iSenhas iOS application.

Security reviews and vulnerability reports should focus on:

* Cryptographic implementation
* Local vault protection
* Key management
* Data handling on device
* Authentication and biometric integration
* Memory exposure risks
* Client-side security logic

The following systems are **out of scope**:

* Backend infrastructure
* Production APIs
* Authentication servers
* Synchronization services
* Internal operational tooling

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

📧 [security@isenhas.com.br](mailto:security@isenhas.com.br)

Please include:

* Description of the issue
* Steps to reproduce
* Potential impact
* Proof of concept (if available)
* Affected version

Avoid public disclosure before coordinated remediation.

---

## Response Process

iSenhas follows a responsible disclosure process:

1. Report received
2. Acknowledgement of receipt
3. Investigation and validation
4. Fix development
5. Security update release
6. Coordinated public disclosure (when appropriate)

We aim to acknowledge reports within **72 hours**.

---

## Security Research Guidelines

Security research is welcome.

Allowed activities include:

* Code review
* Static analysis
* Reverse engineering of the public client
* Cryptographic analysis

Please do NOT:

* Attempt to access other users’ data
* Attack production infrastructure
* Perform denial-of-service testing
* Exploit vulnerabilities publicly

---

## Safe Harbor

Good-faith security research conducted under this policy will not be considered malicious activity.
